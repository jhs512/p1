## 목표

Spring Security를 처음 접하는 코프링 개발자에게 다음 세 가지를 명확히 전달한다:

1. Spring Security가 무엇인지 (보안 프레임워크 역할)
2. 인증(Authentication)과 인가(Authorization)의 차이
3. `build.gradle.kts`에 의존성을 추가하는 실용적 첫 단계

### 5. DependencyScene

**AUDIO_CONFIG 키:** `dependency` → **mp3:** `ss-dependency.mp3`

- 나레이션:
  - ① "코프링 프로젝트에 Spring Security를 추가하려면 [build.gradle.kts(발음:빌드 그래들 케이티에스)]에 의존성을 추가합니다."
  - ② "스타터 하나로 기본 보안 설정이 모두 적용됩니다."
- 비주얼: `build.gradle.kts` 코드 블록 순차 등장 (타이핑 애니메이션 아님 — 단계별 reveal)
  ```kotlin
  dependencies {
      implementation("org.springframework.boot:spring-boot-starter-security")
  }
  ```
- 타이밍 동기화 (모든 anchor는 AUDIO_CONFIG 값, 하드코딩 오프셋 없음):
  - ① `dependencies {` → `speechStartFrame`에 등장 (spring stagger: `durationInFrames: 20`)
  - ② `implementation(...)` 줄 → `narrationSplits[0]`에 등장 (spring stagger: `durationInFrames: 20`)
  - ③ `}` → `narrationSplits[0]`에 등장 (spring stagger: `durationInFrames: 35` — 자연스러운 시차)
- 모노스페이스 폰트에 반드시 `MONO_NO_LIGA` 적용
- 색상 분리:
  - `C_CODE` (`#4e9cd5`) = 키워드 `dependencies`, `implementation`
  - `C_STRING` (`#ce9178`) = 따옴표 포함 전체 문자열 `"org.springframework.boot:spring-boot-starter-security"`
  - 중괄호 `{`, `}` 및 괄호 `(`, `)` = 흰색 (`#ffffff`)
- Duration override (③ spring 완료 프레임 기준):
  ```ts
  const ANIM_END = AUDIO_CONFIG.dependency.narrationSplits[0] + 35;
  const sceneDuration = Math.max(
    AUDIO_CONFIG.dependency.durationInFrames,
    ANIM_END + CROSS + SCENE_TAIL_FRAMES,
  );
  ```
